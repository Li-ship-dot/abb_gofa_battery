## 五次多项式算法原理

```
q(t) = a₀ + a₁t + a₂t² + a₃t³ + a₄t⁴ + a₅t⁵
```

**边界条件（6个约束确定6个系数）:**
- q(0) = q₀ (初始位置)
- q'(0) = qd₀ (初始速度)
- q''(0) = qdd₀ (初始加速度)
- q(T) = q₁ (目标位置)
- q'(T) = qd₁ (目标速度)
- q''(T) = qdd₁ (目标加速度)

# ROS2 架构底层解析

## 第一性原理：ROS2 是什么？

ROS2 不是操作系统，而是一个**中间件（Middleware）**：

```
┌─────────────────────────────────────────────────────────┐
│                      你的应用程序                          │
├─────────────────────────────────────────────────────────┤
│                      ROS2 中间件                          │
│   ┌──────────┐  ┌──────────┐  ┌──────────────────────┐  │
│   │  发布者   │  │  订阅者   │  │  Service / Action    │  │
│   │(Publisher)│  │(Subscriber)│  │  (服务/动作)         │  │
│   └────┬─────┘  └────┬─────┘  └──────────────────────┘  │
│        │              │                                   │
│   ┌────▼──────────────▼─────┐                            │
│   │      DDS 通信层          │  ← 真正的数据传输          │
│   │  (Data Distribution     │                            │
│   │   Service)              │                            │
│   └─────────────────────────┘                            │
├─────────────────────────────────────────────────────────┤
│                  操作系统 (Linux)                         │
└─────────────────────────────────────────────────────────┘
```

### 核心概念：进程间通信

在没有ROS的情况下，两个程序要通信，你需要自己写：
- 共享内存 / Socket / 管道
- 数据序列化/反序列化
- 连接管理

ROS2帮你做了这一切，你只需要：

```python
# 发布者
pub = node.create_publisher(String, 'chat', 10)
pub.publish("Hello")

# 订阅者
sub = node.create_subscription(String, 'chat', callback)
```

## ROS2 节点通信模式

### 1. Topic（发布/订阅）— 异步单向通信

```
                    ┌─────────┐
                    │  Node A  │
                    │ publisher│
                    └────┬────┘
                         │  /joint_states [JointState]
                         ▼
┌──────────────────────────────────────────────────┐
│                   DDS 总线                        │
│  (自动发现、自动连接、多对多通信)                    │
└──────────────────────────────────────────────────┘
                         │
                    ┌────▼────┐
                    │  Node B  │
                    │ subscriber│
                    └─────────┘
```

**特点**：
- 发完即忘，不需要回应
- 多个订阅者可以同时收到
- 用于持续数据流（传感器、状态）

### 2. Service（服务）— 同步请求/响应

```
     ┌─────────┐                    ┌─────────┐
     │  Client │ ─── request ────▶  │  Server │
     │         │ ◀─── response ────│         │
     └─────────┘                    └─────────┘
```

**特点**：
- 有请求必有响应
- 阻塞式
- 用于查询、触发操作

### 3. Action（动作）— 长时间任务

```
     ┌─────────┐      ┌──────────────┐      ┌─────────┐
     │  Client │ ────▶│   Action     │◀─────│  Server │
     │         │◀─────│  Server      │─────▶│         │
     └─────────┘      └──────────────┘      └─────────┘
         │                 │
         │   goal ────────▶│
         │◀── feedback ────│
         │◀── result ──────│
```

**特点**：
- 可以取消
- 有进度反馈
- 用于长时间运行任务（机械臂移动）

## MoveIt2 是什么？

MoveIt2 是一个**运动规划框架**，不是单一节点。

```
┌─────────────────────────────────────────────────────────────────┐
│                         MoveIt2 框架                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│  ┌─────────────┐   ┌─────────────┐   ┌─────────────────────────┐ │
│  │  move_group │   │  planning_pipeline │ │  kinematic_model    │ │
│  │  (核心节点)   │   │  (OMPL/Pilz)       │ │  (运动学模型)         │ │
│  └──────┬──────┘   └─────────────┘   └─────────────────────────┘ │
│         │                                                          │
│  ┌──────▼──────────────────────────────────────────────────────────┐│
│  │              planning_scene_monitor                              ││
│  │  (碰撞检测、场景管理)                                              ││
│  └──────────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────────┘
```

### move_group 节点详解

move_group 是 MoveIt2 的核心进程，它：

1. **接收请求**：
   - 来自 RViz 的 GUI 请求
   - 来自 Python/C++ API 的编程请求
   - 来自其他节点的服务请求

2. **调用规划器**：
   - OMPL（随机采样）：适合复杂场景
   - Pilz（工业机械臂）：适合点到点
   - CHOMP（轨迹优化）：适合平滑路径

3. **碰撞检测**：
   - 使用 FCL 库
   - 检查与障碍物、自碰撞

4. **输出轨迹**：
   - 发布到 `/joint_trajectory`
   - 供 controller 执行

### MoveIt 与 ROS2 的关系

```
RViz (GUI工具)
    │
    │ 1. 用户拖动机械臂
    ▼
/move_group/goal (Action)
    │
    │ 2. 运动规划
    ▼
planning_pipeline (OMPL等)
    │
    │ 3. 生成轨迹
    ▼
/joint_trajectory (Topic)
    │
    ▼
controller_manager
```

## ros2_control 是什么？

ros2_control 是 ROS2 的**硬件抽象层**：

```
┌─────────────────────────────────────────────────────────────┐
│                    ros2_control 架构                         │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  ┌───────────────────────┐      ┌───────────────────────────┐│
│  │  controller_manager   │ ───▶ │  joint_trajectory_controller│
│  │  (控制器管理器)         │      │  (关节轨迹控制器)          ││
│  │                       │      └────────────┬──────────────┘│
│  │  - 加载/卸载控制器      │                   │               │
│  │  - 管理控制器生命周期   │                   │ 位置命令       │
│  │  - 提供服务接口        │                   ▼               │
│  └───────────────────────┘      ┌───────────────────────────┐│
│                                 │  hardware_interface       ││
│  ┌───────────────────────┐      │  (硬件接口)                ││
│  │ joint_state_broadcaster│◀────│  - 读取: 当前位置/速度     ││
│  │ (发布关节状态)         │     │  - 写入: 目标位置          ││
│  └───────────────────────┘      └────────────┬──────────────┘│
└───────────────────────────────────────────────┼───────────────┘
                                                │
                    ┌──────────────────────────┼───────────────┐
                    │                          ▼               │
                    │  ┌──────────────────────────────────────┐│
                    │  │        实际硬件 / 仿真                 ││
                    │  │  ┌─────────┐ ┌─────────┐ ┌─────────┐ ││
                    │  │  │FakeSystem│ │  EGM    │ │  RWS    │ ││
                    │  │  │(模拟)    │ │(实时控制)│ │(Robot   │ ││
                    │  │  └─────────┘ └─────────┘ │ WebSvc) │ ││
                    │  │                         └─────────┘ ││
                    │  └──────────────────────────────────────┘│
                    └──────────────────────────────────────────┘
```

### 为什么需要 ros2_control？

没有 ros2_control 时：
- 每个机器人的控制代码都不同
- 换了机器人，代码全要改

有了 ros2_control 后：
- 统一接口：`hardware_interface`
- 换了机器人，只需换 driver，不改上层代码

### controller_manager 的作用

```cpp
// 伪代码：controller_manager 内部
class ControllerManager {
    // 加载控制器
    void load_controller("joint_trajectory_controller");
    void load_controller("joint_state_broadcaster");

    // 每个控制周期调用
    void update() {
        // 读取硬件状态
        auto state = hw->read();

        // 更新控制器
        controller->update(state);

        // 写入硬件命令
        hw->write(controller->output());
    }
}
```

## 各层职责总结

| 层次 | 组件 | 职责 | 类比 |
|------|------|------|------|
| 应用层 | RViz | 可视化、交互 | 显示器 |
| 应用层 | 你的轨迹节点 | 轨迹平滑 | 后处理滤镜 |
| 规划层 | MoveIt | 运动规划、碰撞检测 | 大脑 |
| 控制层 | ros2_control | 硬件抽象、位置控制 | 神经 |
| 驱动层 | EGM/RWS/Fake | 实际通信/仿真 | 肌肉 |

每一层只知道上下相邻层的接口，不知道其他层的细节。这就是**分层架构**的好处。
